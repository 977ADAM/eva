# Eva — разговорный режим после wake-word

**Дата:** 2026-05-27
**Статус:** draft (ожидает реализации)
**Ветка:** `feat/conversation-mode`

## Цель

Дать пользователю возможность вести с Евой свободный диалог: после первого обращения через wake-word Ева остаётся «открытой» N секунд и реагирует на следующие реплики без необходимости каждый раз начинать с «Ева». По таймауту режим закрывается сам.

## Контекст

Текущая логика [`Assistant.handle_text`](../../eva/assistant.py): если в фразе есть «ева/эва/eva/ява» — обрабатываем, иначе игнорируем. Это вынуждает повторять имя в каждой реплике, что некомфортно при последовательных вопросах («Ева, какая погода» → ответ → «Ева, а завтра?»).

Цель — поведение в духе Яндекс.Алисы (follow-up window) или Siri/Alexa: первый wake-word открывает короткую сессию, в которой можно говорить свободно.

## Решение

Добавить в `Assistant` отслеживание времени конца сессии. Любой обмен (вопрос пользователя или ответ Евы) продлевает сессию ещё на 60 секунд от текущего момента. Если сессия активна — wake-word не требуется. Если сессия истекла или ни разу не открывалась — поведение прежнее (требуется wake-word).

## Архитектура

### Состояние сессии

`Assistant` получает три новых атрибута:

- `conversation_until: float | None` — `time.monotonic()`-таймштамп, после которого сессия истекает; `None` означает «сессии нет».
- `_time_source: Callable[[], float]` — инъекция источника времени (по умолчанию `time.monotonic`), нужна для детерминистских тестов без `time.sleep`.
- `_conversation_timeout: float` — берётся из `Config.conversation_timeout_sec`, хранится локально для удобства.

### Изменения в `handle_text`

Текущий поток (упрощённо):
```
exit_words → sleep/wake_again → if require_wake: need_wake_in_text → strip → ask_brain
```

Новый поток:
```
exit_words → sleep/wake_again → if sleeping: return
  → conversation_active() → strip_wake_word_if_present → ask_brain → extend_session
  → если есть wake_word → strip → ask_brain → open_session
  → иначе игнор
```

Где:
- `conversation_active()` — `self.conversation_until is not None and self._time_source() < self.conversation_until`
- Открытие/продление: `self.conversation_until = self._time_source() + self._conversation_timeout`

Сессия продлевается **один раз за обмен** — в тот момент когда мы решили обработать фразу (после успешной транскрипции, перед отправкой в Brain). 60 секунд отсчитываются от момента когда юзер закончил говорить.

Этого достаточно: типичный ответ Евы 1-5 сек, у юзера остаётся 55+ сек на следующий вопрос. Двойное продление (до и после Brain) усложняет логику без реального выигрыша.

### Sleep mode остаётся главным

Если `sleeping=True`, никаких сессий нет: реплики игнорятся, активная сессия (если была) **не сбрасывается явно**, но при следующем «проснись» окажется истёкшей по времени — это ожидаемо. Sleep — это «выключаюсь временно», и возврат начинается с чистого листа.

### Exit-words имеют приоритет

Если в фразе есть `exit_words` — Eva выключается, неважно активна ли сессия.

### Wake-word в открытой сессии

Если пользователь всё же сказал «Ева» внутри сессии — `_strip_wake_word` его уберёт (логика уже есть). Никакой ошибки или повторной активации не происходит, просто привычка пользователя не мешает.

### Config

Добавляется одно поле:

```python
conversation_timeout_sec: float = 60.0
```

CLI-флаг **не** добавляется — параметр редко нужно крутить, перенастройка через редактирование Config-defaults достаточна. YAGNI.

### Никаких звуковых индикаторов

Не добавляем «бип» при открытии/закрытии сессии. У Евы нет железной подсветки, а голосовое уведомление будет раздражать. Состояние ясно из контекста (только что говорил с Евой → сессия открыта).

## Поток данных

```
utterance from segmenter
   ↓
transcribe
   ↓
handle_text(text)
   ├─ exit → stop
   ├─ sleep/wake_again → toggle sleeping
   ├─ if sleeping → return
   ├─ if conversation_active or text contains wake_word:
   │     extend conversation (one extension at start of exchange)
   │     strip wake_word (no-op if absent)
   │     ask brain → speak deltas
   └─ else: return (ignore)
```

## Что НЕ входит

- LLM-side фильтр «обращено ли это ко мне?» — отдельный LLM-вызов на каждую фразу, дорого и медленно. При появлении проблемы можно ввести вторым итерацией.
- Голосовая команда «закрыть разговор» сейчас — таймаут и существующее sleep-word достаточны. Можно добавить позже одной строкой в spec.
- Поддержка нескольких параллельных сессий — Ева для одного пользователя.
- Звуковой индикатор / лог открытия сессии — debug-лог через стандартный `logging` бесплатно даст это без UX-шума.

## Обработка ошибок

| Сценарий | Поведение |
|---|---|
| Brain отдал error в сессии | Eva говорит «Связь пропала». Сессия уже была продлена (мы продлеваем до вызова Brain) — юзер может попробовать ещё раз без wake-word в течение таймаута. |
| Eva не успела озвучить и зависла | Сессия истечёт по таймауту, ничего ломаться не будет |
| Пользователь сказал что-то в сессии, но Whisper выдал пусто | Игнор как и сейчас; сессия не продлевается (нет реального обмена) |

## Тестирование

Расширяем [`tests/test_assistant_logic.py`](../../tests/test_assistant_logic.py) пятью новыми тестами с инъекцией фейкового `time_source`:

1. **`test_in_session_no_wake_word_processed`** — после первого «Ева, X» сразу следующая фраза без «Ева» уходит в Brain.
2. **`test_each_exchange_extends_session`** — после двух обменов сессия закрывается через timeout от последнего, а не от первого.
3. **`test_session_expires_after_timeout`** — продвигаем фейковое время > timeout — фраза без wake-word снова игнорится.
4. **`test_sleeping_blocks_session_use`** — `sleeping=True`, фраза без wake-word внутри активной сессии игнорится (sleep > session).
5. **`test_brain_error_still_extends_session`** — Brain отдал `ResponseDelta(error=...)` — `conversation_until` всё равно сдвинулся (продление до Brain), юзер может попробовать ещё раз без wake-word.

Существующие 11 тестов остаются зелёными: для них `time_source` дефолтный, сессия не открыта, поведение прежнее.

## Критерии приёмки

1. После «Ева, как дела» → ответ → «А завтра погода?» — Ева отвечает (раньше игнорировала бы).
2. Помолчал 60+ сек → фраза без «Ева» снова игнорится.
3. «Ева, замолчи» во время сессии — Ева молчит до «Ева, проснись» (sleep работает как раньше).
4. «Ева, выключись» во время сессии — Eva корректно завершается.
5. Все 16 unit-тестов зелёные (`uv run pytest` — 11 старых + 5 новых).
6. Полная функциональность сохраняется при `--no-wake` (там просто wake-word не проверяется, а conversation_timeout не имеет смысла, но и не мешает).
